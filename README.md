# Sewers & Shadows — Float Chess Discord Bot

A duet chess-variant bot. Two modes: play Float Chess against
Pennywise yourself, or watch an autonomous Violet-vs-Pennywise
exhibition match. Deliberately split into layers that don't trust each
other more than they have to:

```
float-chess-engine.js   — pure rules engine, no persona/Discord knowledge.
                           Owns legality, resolution, check/checkmate.
                           (lives one directory up — shared, reusable
                           outside this bot if you ever want it.)

src/pennywiseAI.js       — picks Pennywise's MOVE. Deterministic-ish
                           heuristic evaluator, no LLM involved. If this
                           picked bad moves, at least they'd be legal ones.

src/violetAI.js          — picks Violet's MOVE. Same architecture as
                           pennywiseAI.js, deliberately different
                           character: checks whether a move would leave
                           her own piece capturable next turn (methodical,
                           not flashy), less random jitter.

src/personaEngine.js     — shared factory: loads a system prompt, calls
                           the Anthropic API, turns engine events into
                           plain-English context. No character knowledge
                           of its own — see the two files below for that.

src/pennywisePersona.js  — Pennywise's voice (config/pennywise-system-prompt.md)
src/violetPersona.js     — Violet's voice (config/violet-system-prompt.md)
                           Neither persona module ever touches game state —
                           they only narrate what the engine says already
                           happened. If the LLM says something dumb, worst
                           case is a weird line of dialogue, not a
                           corrupted board.

src/gameManager.js       — per-channel session state (two modes — see
                           below) + simple stats persistence (data/stats.json).

src/index.js             — Discord.js slash commands & message handling.
```

The move/dialogue split matters: an LLM hallucinating an illegal chess
move is a real problem (silently corrupts game state, or requires the
engine to reject and awkwardly re-prompt). An LLM hallucinating a
slightly off line of flavor text is not. Keeping move-selection and
narration as separate calls means the worst a bad LLM response can do
is sound a little off — it can never break the game.

## Two modes

- **`vs-pennywise`** — a human plays white against Pennywise (AI,
  black) via `/move`. Wins/losses/draws are tracked per-user.
- **`exhibition`** — fully autonomous: Violet (white, AI) vs Pennywise
  (black, AI) play each other out, one turn every 6 seconds, with both
  personas narrating from their own point of view each turn — Violet
  clinical and dry, Pennywise theatrical, deliberately contrasting
  registers. `/resign` stops it early. No stats are recorded (there's
  no human player in this mode).

Pick the mode with `/newgame mode:`.

## Setup

```bash
cd bot
npm install
cp .env.example .env
# fill in DISCORD_TOKEN, DISCORD_CLIENT_ID, ANTHROPIC_API_KEY
npm start
```

You'll need a Discord application + bot token from the
[Discord Developer Portal](https://discord.com/developers/applications),
invited to your server with the `applications.commands` and `bot`
scopes (Send Messages, Use Slash Commands permissions are enough).

## Commands

- `/newgame [mode:]` — starts a game in the current channel (one game
  per channel at a time); mode defaults to `vs-pennywise`
- `/move from:e2 to:e4 [special:] [promotion:]` — submit your move
  (vs-pennywise mode only)
- `/rules` — posts the actual ruleset (kept in `src/rulesText.js`,
  describing what the engine enforces — not generated by the LLM, so
  it can't drift from what actually happens)
- `/resign` — forfeit (vs-pennywise) or stop the show early (exhibition)
- `/stats` — your win/loss/draw record
- `/song` — link to Pennywise's source track
- Typing `RED BALLOON` in all caps drops Pennywise's intensity
  immediately (safe word, handled in code before any LLM call — see
  index.js). Only active in vs-pennywise channels.

## Known limitations / things to revisit before this is "done"

- **Public move submission.** `/move` is a public slash command for
  simplicity. Since Pennywise's move is chosen from the pre-turn board
  *before* looking at your submission (see
  `gameManager.submitHumanMoveAndResolve`), the vs-pennywise matchup
  stays genuinely simultaneous even with public moves. If you extend
  this to real human-vs-human duets, switch move submission to DMs so
  opponents can't see each other's pending move.
- **AI move quality.** Both `pennywiseAI.js` and `violetAI.js` are
  1-ply heuristic evaluators — no real lookahead search. They play
  reasonably (Violet noticeably more cautiously — see her
  hanging-piece check) but neither is a strong chess engine. Swap the
  evaluator if you want tougher opponents; nothing else in the stack
  needs to change.
- **Multi-king checkmate edge case.** If Crossing the Streams ever
  gives a color two Kings, the engine's checkmate logic uses a
  conservative reading (flagged inline in float-chess-engine.js as a
  `DESIGN CALL`) — worth a second look if that scenario comes up in
  actual play.
- **Stats persistence** is a flat JSON file, fine for one bot process,
  not for horizontal scaling. Swap `gameManager.js`'s
  `loadStats`/`saveStats` for a real datastore if this grows.
- **Exhibition mode has no pause/resume**, only stop-and-discard via
  `/resign`. Worth adding if long unattended matches turn out to be
  something people want to leave running and check back on.
- **Neither persona's tone is built from the actual song's lyrics** —
  see the `TONAL SOURCE` section of `config/pennywise-system-prompt.md`
  for why, and don't relax that even if it feels safe to.
